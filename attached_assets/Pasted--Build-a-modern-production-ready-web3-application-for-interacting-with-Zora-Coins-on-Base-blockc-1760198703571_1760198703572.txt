#### POST /api/auth/verify-signature
**Purpose**: Verify signature and get JWT token  
**Auth**: None  
**Body**:
```json
{
  "walletAddress": "0x...",
  "signature": "0x...",
```

**Response**:
```json

  "success": true,
  "message": "Authentication successful",
  "data": {
    "token": "eyJhbG...",
    "user": {
      "walletAddress": "0x...",

  }

```

#### POST /api/auth/refresh
**Purpose**: Refresh expired JWT token  
**Auth**: JWT Required  
**Response**: New JWT token

#### POST /api/auth/logout
**Purpose**: Invalidate current session  
**Auth**: JWT Required

**UI Flow**:
1. Connect wallet via RainbowKit
2. Get nonce for wallet address
3. Sign message with wallet (SIWE standard)
4. Verify signature to get JWT
5. Store JWT in localStorage/sessionStorage
6. Include JWT in Authorization header for protected routes

## ğŸ“Š Data Fetching Strategy

### React Query Setup

```typescript
// Use TanStack Query for all API calls
// Benefits: Caching, auto-refetch, loading states, error handling

// Example query keys:
['coin', address]
['coins', 'explore', 'top-gainers', { after, count }]
['profile', identifier]
['scheduled-coins', { status, limit, skip }]
['trade-quote', { sell, buy, amountIn }]
```

### Fetching Patterns

**Queries** (GET requests):
- Auto-fetch on component mount
- Cache for 5 minutes
- Refetch on window focus
- Stale time: 1 minute

**Mutations** (POST/PUT/DELETE):
- Invalidate related queries on success
